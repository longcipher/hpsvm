## Code Conventions

- **Naming:** snake_case for functions/variables, PascalCase for types, SCREAMING_SNAKE for constants
- **Error Handling:** `thiserror` for library errors, `HPSVMError` enum in `crates/hpsvm/src/error.rs`
- **Concurrency:** `parking_lot` for synchronous locks, `thread::scope` for batch parallelism
- **Import Style:** grouped by crate, `use` imports at module top
- **Linting:** `clippy::pedantic` + `clippy::nursery` enabled workspace-wide, `unwrap_used = "deny"`
- **Formatting:** `cargo +nightly fmt`, `cargo sort`, `rumdl` for markdown

## Architectural Decisions (Existing)

- HPSVM struct has 17+ fields with manual `Clone`/`Debug` impls
- Batch execution uses `thread::scope` with per-worker `AccountsDb` clones
- Deferred commit model: `transact` returns `ExecutionOutcome`, `commit_transaction` applies it
- External `AccountSource` trait for pluggable account backends
- `register_tracing` feature for BPF register trace persistence
